## 1. 简介

### 1.1 YCL 是什么？

**Your Craft Launcher（简称 YCL）** 是一款专门用来启动《我的世界》Java 版（Minecraft: Java Edition）的小工具。你可以把它理解成"启动游戏的遥控器"。

它能帮你做这些事：

- 一键启动游戏，不用自己写命令行
- 下载和安装各种游戏版本（比如 1.20.4、1.18.2 这些）
- 安装模组加载器（Forge、Fabric、Quilt、NeoForge、LiteLoader）
- 在线搜索和下载模组、整合包、资源包、光影包、地图
- 管理多个账户（离线、微软正版、外置登录）
- 自动找到你电脑上装好的 Java，也可以一键下载新 Java
- 备份和恢复你的游戏存档
- 管理你在游戏里截的图
- 切换下载源，国内下载更快
- 检查启动器有没有新版本

### 1.2 技术栈是什么？（这节给好奇的同学看）

> "技术栈"就是"做这个软件用到的工具组合"。就像做蛋糕要面粉、鸡蛋、糖一样，做软件也要用各种"材料"。

- **.NET 10**：微软做的开发平台，YCL 就是基于它做的。
- **WPF**：用来画窗口界面的工具，可以让窗口看起来很漂亮。
- **iNKORE.UI.WPF.Modern**：一套让窗口变成 Windows 11 风格的"皮肤"。
- **C#**：一种编程语言，YCL 全是用它写的。

### 1.3 软件界面长啥样？

打开 YCL 后，你会看到左边有一排菜单（像主页、启动、下载这些按钮），右边是显示内容的地方。点哪个菜单，右边就显示对应的内容。

## 2. 安装步骤

### 2.1 你的电脑要满足什么条件？

YCL 能在大部分 Windows 电脑上跑，但有几个小要求：

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 1809 版本或更高（推荐 Windows 11） |
| 系统位数 | 必须是 64 位系统 |
| .NET 运行时 | .NET 10 桌面运行时（安装包会自动帮你装） |
| 内存 | 至少 4GB（玩 Minecraft 至少要 4GB，建议 8GB 以上） |
| 硬盘空间 | 至少 500MB 给启动器，外加游戏本身需要 2~10GB |
| 网络 | 需要联网下载游戏文件 |

> **小贴士**：怎么看你 Windows 是几版本？
> 按 `Win + R` 键，输入 `winver` 回车，会弹出一个小窗口告诉你版本号。

### 2.2 两种安装方式

#### 方式一：用安装包（推荐新手）

1. 拿到 `YCL-Setup-1.0.0.exe` 这个安装文件。
2. 双击运行它。
3. 跟着向导走：
   - 选择"我同意"许可协议（其实没什么协议，就是别拿去做坏事）
   - 选择安装位置（不知道选什么就保持默认，装到 `C:\Program Files\YCL`）
   - 勾选"创建桌面快捷方式"（方便以后找）
   - 点"安装"
4. 等几秒钟，安装完点"完成"。
5. 你可以在桌面或开始菜单找到"Your Craft Launcher"。

#### 方式二：免安装绿色版（推荐老手）

1. 拿到 YCL 的发布文件夹（`publish` 文件夹，里面有一堆 DLL 和一个 `YCL.exe`）。
2. 把它复制到你想放的地方，比如 `D:\YCL\`。
3. 双击 `YCL.exe` 就能运行，不用安装。

> 这种方式的好处：可以直接复制到 U 盘带着走，不用每次都装。坏处：没有桌面快捷方式，要自己右键创建一个。

### 2.3 第一次启动要注意什么？

第一次打开 YCL，它会做这几件事：

1. **自动创建配置文件**：在 `C:\Users\你的用户名\AppData\Roaming\YCL\` 这个文件夹里创建 `config.json`，存你的设置。
2. **创建日志文件夹**：在同目录下创建 `logs\` 文件夹，里面记着启动器每天干的事。
3. **可能提示你装 .NET**：如果电脑里没有 .NET 10 运行时，会跳出来一个微软的下载链接，按提示装一下就行。

> **小知识**：`AppData` 这个文件夹默认是隐藏的。想看到它，需要在文件资源管理器的"查看"里勾选"隐藏的项目"。

打开后，你会看到默认是"主页"。别急着点启动，**第一次用要先设置游戏目录和 Java**，下面会讲怎么做。

## 3. 功能说明

左边导航栏从上到下一共有 11 个页面。下面一个一个讲。

### 3.1 主页（版本管理）

**主页是管理 Minecraft 游戏版本的地方。**

你会看到一个列表，显示你电脑上已经装好的所有版本（比如 1.20.4、1.18.2-Forge 这些）。每个版本旁边有几个按钮：

- **安装新版本**：点这个按钮，会显示所有官方版本列表（包括正式版 Release、快照 Snapshot、远古版本 Old Alpha 等）。选一个版本点安装，YCL 会自动下载所有需要的文件。
- **删除**：把选中的版本从电脑上删掉。**注意**：会弹出一个"你确定要删吗？"的对话框，因为删了就找不回来了。
- **重命名**：给版本改个名字，方便记住（比如改成"我的世界 1.20.4 装了模组版"）。
- **复制**：把现有版本复制一份，做成新版本。如果你想试一个新模组又怕弄坏原版，就可以先复制一份。
- **打开文件夹**：直接打开这个版本所在的文件夹，方便看里面的文件。
- **安装/卸载加载器**：给这个版本装模组加载器（比如 Fabric、Forge）。下面详细讲。

> **什么是版本？**
> Minecraft 每个版本（比如 1.20.4、1.18.2）都有自己的一套文件。装了 Forge 的 1.20.4 和没装的 1.20.4 是两个不同的"版本"，YCL 会把它们当成两个东西来管理。

### 3.2 启动

**这是你最常用的页面，用来真正启动游戏。**

页面里有：

- **版本选择下拉框**：选一个你想玩的版本。
- **账户显示**：显示当前用的是哪个账户。
- **启动按钮**：大大的按钮，点了就开始启动游戏。启动时会显示"准备中..."、"启动中..."这种状态。
- **日志面板**：游戏启动后，下面的方框会实时显示游戏输出的日志（就是游戏一边跑一边打印出来的信息）。如果游戏崩了，这里会有错误信息。
- **内存设置**：可以调游戏最多用多少内存（默认 2048MB = 2GB）。如果游戏卡，可以调大点；如果电脑内存不够，调小点。

> **启动后会怎么样？**
> YCL 会先检查所有游戏文件是不是完整，缺了就自动下载；都齐全后才会启动 java 进程打开游戏。所以第一次启动可能慢一点，之后就快了。

### 3.3 下载

**这是看你下载进度的地方。**

当你点了"安装版本"或者下载模组时，下载任务会出现在这里。每个任务显示：

- **文件名**：正在下载什么文件
- **进度条**：下载到百分之多少了
- **速度**：每秒下载多少 KB / MB
- **暂停 / 继续**：临时停下来，之后再继续（断点续传，不会重头开始）
- **取消**：彻底取消这个下载任务

下载源可以在设置里切换（官方源 / BMCLAPI 镜像 / MCBBS 镜像）。国内推荐用 BMCLAPI，快很多。

### 3.4 模组

**这是管理游戏模组（Mods）的地方。模组就是给游戏加新东西的小插件。**

#### 本地模组管理

上半部分显示你已经放进 `mods` 文件夹里的所有模组，每个模组显示：

- 图标（如果有）
- 模组名字、版本、说明
- 当前状态（启用 / 禁用）

操作：

- **启用 / 禁用**：点一下就能切换状态。禁用就是把文件改名成 `.disabled`，不删文件，下次还能再启用。
- **删除**：彻底删掉这个模组文件。
- **打开 mods 文件夹**：方便你手动往里塞模组。

#### 在线搜索模组

下半部分可以从网上搜模组：

- 输入框里打模组名字（比如"JEI"）
- 选择搜索源：Modrinth 或 CurseForge
- 点搜索，会出来一堆结果
- 每个结果显示名字、作者、下载量、简介
- 点下载，会自动放到当前版本的 `mods` 文件夹里

> **小知识**：Modrinth 是公开的，不用 Key 就能搜。CurseForge 要在设置里填 API Key 才能搜（Key 怎么拿看 FAQ）。

### 3.5 资源

**这里可以下载各种 Minecraft 资源：**

- **整合包**：别人配好的全套模组+配置，下载下来就是一个完整的游戏体验
- **资源包**：改变游戏的画面和声音（比如让草地看起来更真实）
- **光影包**：让游戏的光线更好看（需要 OptiFine 或 Iris 等支持）
- **地图**：别人做好的地图，下载下来就能玩

操作和模组页差不多：选类型 → 输入关键词 → 搜索 → 下载。

下载整合包时，YCL 会自动帮你装好需要的 Minecraft 版本和所有模组，不用你自己一个个装。

### 3.6 账户

**这里管你的游戏账户。** 一个启动器里可以存多个账户，方便切换。

有三种账户可以加：

#### 离线账户

- 最简单的账户，不用联网，不用密码
- 直接输入一个用户名就能用
- 缺点：不能进需要正版验证的服务器
- 适合：自己一个人玩、或者跟朋友联机（朋友也用离线账户）

#### 微软账户（正版）

- 用你买 Minecraft 的微软账号登录
- 登录方式叫"设备代码流"（Device Code Flow），步骤是：
  1. YCL 会显示一个 8 位数的代码（比如 `ABCD-EFGH`）
  2. 浏览器会自动打开微软登录页（或者你手动打开 `https://microsoft.com/link`）
  3. 输入那 8 位数代码，登录你的微软账号
  4. 等几秒钟，YCL 会自动拿到登录信息
- 登录一次后，下次启动器会自动刷新登录，不用再登一遍
- 能进所有需要正版验证的服务器

#### 外置账户（authlib-injector）

- 用来登录"第三方认证服务器"
- 比如有些正版服务器（像"皮肤站"）就是用这种方式
- 需要填一个"认证服务器地址"（一个 https 开头的网址）
- YCL 会自动下载 authlib-injector 这个工具，第一次会有点慢

每个账户卡片会显示：

- 玩家名字
- 头像和皮肤（如果有）
- 账户类型
- 上次刷新时间

操作：

- **添加账户**：选好类型，按提示操作
- **切换账户**：点一下账户，把它设为当前使用的账户
- **删除账户**：删掉这个账户
- **刷新**：重新登录一下，更新令牌（令牌过期了就需要刷新）

### 3.7 Java

**Java 是 Minecraft 运行必须的东西。Minecraft 是用 Java 语言写的，所以必须有 Java 才能玩。**

页面显示：

- 你电脑上所有检测到的 Java 版本列表
- 每个 Java 显示：路径、版本号（比如 17.0.1、8u301）
- 哪个是"默认 Java"（启动游戏用的就是这个）

操作：

- **自动检测**：YCL 会扫描你电脑上常见的位置（Program Files、注册表等）找 Java
- **手动添加**：如果你知道 Java 装在哪了，可以手动指给 YCL 看
- **安装新 Java**：从 Adoptium（一个免费的 Java 提供网站）下载并安装新 Java。选好版本（比如 17、21）点安装，YCL 会自动下好放到 `.minecraft/runtime/` 目录里
- **设为默认**：选一个 Java 当作启动游戏用的默认 Java

> **小知识**：Minecraft 不同版本需要的 Java 版本不一样：
> - 1.17 以下：要 Java 8
> - 1.17 到 1.17.1：要 Java 16
> - 1.18 到 1.20.4：要 Java 17
> - 1.20.5 以上：要 Java 21
>
> YCL 不会自动判断这个，要你自己选对的版本。

### 3.8 存档

**这里管你的游戏存档（Save）。存档就是你在游戏里建的世界。**

页面显示你 `saves` 文件夹里的所有存档，每个显示：

- 存档名字
- 大小（多大）
- 最后修改时间
- 缩略图（如果有 icon.png）

操作：

- **备份**：把选中的存档打包成一个 zip 文件，存到 YCL 的备份目录。这样你玩坏了还能恢复。
- **恢复**：从备份里恢复一个存档。
- **导入**：把别人给你的存档 zip 文件塞进游戏里。
- **导出**：把你的存档打包成 zip，可以分享给朋友。
- **删除**：删掉这个存档（同样会问你确认）。

> **重要**：玩之前最好备份一下存档！万一模组把世界弄坏了，还能恢复。

### 3.9 截图

**这里管你在游戏里截的图。在游戏里按 F2 就能截图。**

页面显示：

- 所有截图的缩略图（小图）
- 每张截图的名字、时间、大小

操作：

- **查看**：点一张图看大图
- **删除**：删掉不要的截图
- **打开文件夹**：打开截图所在的文件夹，方便复制到别的地方

### 3.10 设置

**这里管 YCL 自己的设置。** 设置分五大类：

#### 启动设置

- 最大内存（默认 2048MB）
- 初始内存（默认 512MB）
- 游戏窗口宽度（默认 854）
- 游戏窗口高度（默认 480）
- 是否全屏启动
- 启动游戏后是否自动关启动器
- 额外的 JVM 参数（高级玩家用，可以加 `-XX:+UseG1GC` 这种）
- 启动前是否清理临时文件

#### 游戏设置

- .minecraft 路径：你的游戏装在哪
- 是否启用版本隔离（开了之后每个版本有自己的 mods/saves 文件夹）

#### 下载设置

- 下载源：官方 / BMCLAPI / MCBBS（国内推荐 BMCLAPI）
- 下载并发数：同时下载几个文件（默认 8，电脑快可以调大）
- 重试次数：下载失败自动重试几次（默认 3）

#### 主题设置

- 主题模式：跟随系统 / 亮色 / 暗色
- 强调色：自定义窗口的颜色

#### 系统设置

- 启动时自动检查更新（默认开）
- GitHub 仓库地址（默认 `YCL-Team/YCL`）
- 开机自启动

> **小贴士**：改了设置别忘了点"保存"按钮，否则下次启动就丢了。

### 3.11 关于

**这里显示 YCL 自己的信息。**

- 版本号（当前 1.0.0）
- 作者、版权信息
- GitHub 链接（可以去看源代码）
- "检查更新"按钮：手动点一下，看看有没有新版本

## 4. 使用指南

### 4.1 第一次用 YCL？跟着这个流程走

第一次用 YCL，按这个顺序操作最顺：

```
1. 安装 YCL → 2. 设置游戏目录 → 3. 装 Java →
4. 加账户 → 5. 装版本 → 6. 点启动！
```

详细步骤：

#### 第一步：设置 .minecraft 目录

1. 打开 YCL，进入"设置 → 游戏设置"。
2. 在".minecraft 路径"那一栏，点"浏览"按钮。
3. 选一个文件夹当你的游戏根目录。建议新建一个空文件夹，比如 `D:\Minecraft\.minecraft`，**别**用别的启动器已经用过的目录，免得冲突。
4. 点"保存"。

> **什么是 .minecraft？**
> 这是 Minecraft 默认的游戏文件夹名字，里面装着所有版本、存档、模组、截图等。所有启动器都认这个名字。

#### 第二步：装 Java

1. 进入"Java"页。
2. 点"自动检测"。如果电脑里以前装过 Java，列表里会出现。
3. 如果一个都没检测到，或者版本不对（比如你的版本需要 Java 17 但只有 Java 8）：
   - 点"安装新 Java"
   - 选需要的版本（比如 17）
   - 点"安装"，等下载完成
4. 选中你想用的 Java，点"设为默认"。

#### 第三步：加一个账户

1. 进入"账户"页。
2. 点"添加账户"，选类型：
   - 自己玩、不需要正版验证：选"离线账户"，输入名字就行
   - 想进正版服务器：选"微软账户"，按提示操作
3. 添加完，点一下账户把它设为当前账户。

#### 第四步：装一个 Minecraft 版本

1. 进入"主页"。
2. 点"安装新版本"。
3. 在版本列表里找一个你想要的版本（推荐正式版 Release，比如 1.20.4）。
4. 点"安装"。YCL 会自动下载版本 JSON、客户端 jar、所有库文件、所有资源文件。可能要等几分钟到几十分钟（看网速）。
5. 下载进度可以在"下载"页看到。

#### 第五步：启动游戏

1. 进入"启动"页。
2. 在版本下拉框里选刚装好的版本。
3. 检查账户对不对。
4. 点"启动"按钮！
5. 等待几秒到几十秒，游戏窗口会弹出来。

### 4.2 怎么装 Forge / Fabric 这些模组加载器？

想玩带模组的版本，必须先装"模组加载器"（ModLoader）。步骤：

1. 进入"主页"。
2. 选中你想装加载器的版本（比如 1.20.4）。
3. 点"安装加载器"按钮。
4. 选加载器类型：
   - **Fabric**：最轻量，启动快，1.14 以上推荐
   - **Forge**：最老牌，支持模组最多，1.13 以上推荐
   - **Quilt**：Fabric 的分叉，能兼容 Fabric 模组
   - **NeoForge**：Forge 的分叉，1.20+ 比较活跃
   - **LiteLoader**：很老的加载器，只支持 1.12 以下
5. 选加载器版本（推荐选最新稳定版）。
6. 点"安装"，等下载完成。
7. 安装完成后，主页会出现一个新版本（比如 `1.20.4-Fabric-0.15.7`），它就是带加载器的版本。

> **要卸载加载器？** 选中带加载器的版本，点"卸载加载器"即可。这会删掉那个版本目录，所以先确认你不需要里面的存档。

### 4.3 怎么下载模组？

1. 先按上面步骤装好模组加载器（比如 Fabric）。
2. 启动一次游戏，让它生成 `mods` 文件夹。如果没生成，自己手动建一个也行。
3. 进入"模组"页。
4. 在"在线搜索"那里：
   - 输入模组名字（比如"JEI"——这是一个看合成表的模组）
   - 选搜索源（Modrinth 推荐，不用 Key）
   - 点"搜索"
5. 在结果里找到想要的，点"下载"。
6. 模组文件会自动放到当前版本的 `mods` 文件夹里。
7. 启动游戏，就能用新模组了！

> **重要**：下载模组要选跟你 Minecraft 版本和加载器匹配的版本！比如 1.20.4 + Fabric 的模组，不能放到 1.18.2 + Forge 里用，会崩。

### 4.4 怎么备份存档？

1. 进入"存档"页。
2. 选中你想备份的存档。
3. 点"备份"。
4. 等几秒钟（存档越大越久），完成会有提示。
5. 备份文件存在 YCL 的备份目录里（`%AppData%\YCL\saves_backup\`）。

要恢复的话：

1. 进入"存档"页。
2. 点"恢复"。
3. 选一个备份文件。
4. 确认覆盖。

### 4.5 怎么切换下载源？

国内访问 Mojang 官方下载源很慢，切换到 BMCLAPI 镜像快很多：

1. 进入"设置 → 下载设置"。
2. "下载源"那里下拉，选"BMCLAPI"。
3. 点"保存"。
4. 之后所有下载都用 BMCLAPI 了。

> **什么是 BMCLAPI？**
> 它是 bangbang93 维护的国内 Minecraft 资源镜像，跟官方源内容一样，但是放在国内服务器上，所以下载快得多。
> MCBBS 镜像现在跟 BMCLAPI 是同一个东西，所以两个选哪个都行。

### 4.6 版本隔离是什么？要不要开？

**版本隔离** = 让每个版本用自己独立的 mods、saves、configs 文件夹。

- **不开版本隔离**（默认）：所有版本共享同一份 mods 和 saves。比如你 1.20.4 装了 10 个模组，1.18.2 也会有这 10 个模组（但版本不对会崩）。
- **开了版本隔离**：每个版本用自己独立的文件夹。1.20.4 装的模组不会影响 1.18.2。

**建议**：如果你玩很多个版本，或者装很多模组，**一定要开**版本隔离！

开启方法：

1. 进入"设置 → 游戏设置"。
2. 勾选"启用版本隔离"。
3. 点"保存"。

开启后，每个版本的游戏目录会变成 `.minecraft/versions/版本名/`，里面有自己的 `mods`、`saves`、`configs` 等子目录。

> 注意：开了版本隔离后，已经存在的存档和模组可能需要手动搬过去。比如你想让旧存档在新版本里玩，要把它从 `.minecraft/saves/` 复制到 `.minecraft/versions/版本名/saves/`。

### 4.7 外置登录（authlib-injector）怎么配？

外置登录可以登录第三方认证服务器，比如某些皮肤站。步骤：

1. 进入"账户"页。
2. 点"添加账户"，选"外置账户"。
3. 填入认证服务器地址（这是服务器给你的一个 https 网址，比如 `https://example.com/api/yggdrasil`）。
4. 输入你的账号和密码。
5. 点"添加"。

YCL 会：

1. 自动下载 `authlib-injector.jar`（第一次会有点慢，存到 `%AppData%\YCL\` 下次就不用再下了）。
2. 预取服务器元数据（让启动更快）。
3. 启动游戏时自动加上注入参数，让游戏用这个服务器来验证身份。

> **小知识**：authlib-injector 用 Java 的"agent"机制，拦截了游戏里的登录请求，重定向到第三方服务器。这样就能用第三方账号玩正版了。

## 5. 常见问题（FAQ）

### Q1：点启动按钮没反应 / 启动失败怎么办？

按这个顺序排查：

1. **Java 路径对吗？** 进入"Java"页，确认默认 Java 选了。如果列表空，先检测或安装。
2. **Java 版本对吗？** 不同 Minecraft 版本要不同 Java（看 3.7 节那个表）。1.20.5 以上要 Java 21，别用 Java 8。
3. **内存够吗？** 进入"设置 → 启动设置"，把最大内存调小一点试试（比如 1024MB）。如果电脑只有 4GB 内存，别开 4096MB。
4. **文件完整吗？** 在"下载"页看看有没有失败的下载。如果有，重新下载。
5. **看日志**：启动页下面的日志面板里看错误信息，或者打开今天的日志文件：
   `C:\Users\你的用户名\AppData\Roaming\YCL\logs\log_2026-07-17.txt`
   里面会有详细错误。

### Q2：下载特别慢怎么办？

切换下载源到 BMCLAPI：

1. 进入"设置 → 下载设置"。
2. 下载源选"BMCLAPI"。
3. 保存。
4. 重新下载。

如果还是慢：

- 检查网络是不是断了
- 关掉防火墙、杀毒软件试试（有时候它们会拦截下载）
- 把下载并发数调小一点（设置 → 下载设置 → 下载并发数改成 4）

### Q3：微软账户登录失败怎么办？

可能的原因：

1. **网络问题**：微软登录服务器在国外，国内访问可能不稳定。可以试试开代理。
2. **代码过期**：设备代码 15 分钟就过期。如果超时了，重新发起一次登录。
3. **没买 Minecraft**：登录成功不等于能玩。你的微软账号必须**买了 Minecraft Java 版**才能进游戏。没买的话，登录成功但启动游戏会失败。
4. **令牌过期**：很久没登录的话，令牌可能过期了。进"账户"页，点"刷新"按钮重新登录。

如果还是不行，看看日志文件里有没有具体错误信息。

### Q4：在模组页搜模组搜不到结果？

可能原因：

1. **CurseForge 没配 API Key**：
   - CurseForge 要求请求带 API Key。
   - 去 https://console.curseforge.com 注册一个开发者账号（免费），创建一个项目，拿到 API Key。
   - 把 Key 填到 YCL 的"设置 → 下载设置 → CurseForge API Key"里。
   - 保存，重新搜索。
2. **Modrinth 也搜不到**：可能是网络问题，或者你搜的关键词太冷门。换个关键词试试。
3. **没选对版本**：搜索时记得选 Minecraft 版本过滤，否则可能搜出来但不兼容。

### Q5：启动器怎么更新？

有两种方式：

#### 自动检查更新

YCL 启动时会自动检查更新（如果设置里开了"启动时检查更新"）。发现新版本会弹窗：

```
发现新版本 1.0.1！
发布时间：2026-07-20
更新内容：...

是否前往下载页面？
```

点"是"会打开浏览器跳到 GitHub Release 页面，自己下载新版本安装包再装一遍就行。

#### 手动检查更新

进入"关于"页，点"检查更新"按钮。

#### 关闭自动检查

如果不想每次启动都弹窗：

1. 进入"设置 → 系统设置"。
2. 取消勾选"启动时检查更新"。
3. 保存。

### Q6：日志文件在哪？

日志文件存在：

```
C:\Users\你的用户名\AppData\Roaming\YCL\logs\
```

每天的日志单独一个文件，名字像 `log_2026-07-17.txt`。

如果你要找 YCL 团队帮忙排查问题，把这个文件夹里**最近几天**的日志文件打包发给他们就行。

### Q7：配置文件在哪？能手动改吗？

配置文件是：

```
C:\Users\你的用户名\AppData\Roaming\YCL\config.json
```

里面是 JSON 格式，可以用记事本打开看。**可以手动改**，但是：

- 改之前最好关掉 YCL（避免被覆盖）
- 改完要保存为 UTF-8 编码（用记事本另存为时选 UTF-8）
- 改坏了不要紧，删掉这个文件，YCL 会用默认配置重建

### Q8：账户令牌保存在哪？安全吗？

账户信息存在：

```
C:\Users\你的用户名\AppData\Roaming\YCL\accounts.json
```

**令牌是加密的！** YCL 用 Windows 的 DPAPI 加密所有敏感令牌（access_token、refresh_token 等），只有你的 Windows 账号能解开。所以：

- 别人复制走这个文件，到他的电脑上是解不开的
- 你换了 Windows 账号登录，可能也解不开（DPAPI 跟用户绑定）

> **小知识**：DPAPI 是 Windows 自带的加密工具，全称 Data Protection API。YCL 用 CurrentUser 范围加密，意思就是"只有当前 Windows 用户能解开"。

### Q9：能同时在多个版本里玩吗？

可以！YCL 启动游戏后会留在后台，你可以再启动另一个版本。但注意：

- 每个游戏实例会占一份内存
- 如果版本不开隔离，存档是共享的（一边玩另一边存档会冲突）
- 不建议同时开太多，电脑会卡

### Q10：能在 Mac 或 Linux 上用吗？

**不能直接用。** YCL 是 WPF 写的，WPF 只支持 Windows。Mac 和 Linux 想玩 Minecraft Java 版，可以用：

- macOS / Linux 上的官方启动器
- MultiMC、PrismLauncher 这些跨平台启动器

如果你想在 Mac/Linux 上跑 YCL，理论上可以通过 Wine 或 Bottles 转译，但**不保证能用**，YCL 团队也不提供支持。
